## Root cause

The admin builder had two separate light-mode issues:

1. `FamilyTreeBuilder.tsx` had already started moving away from dark-only component chrome, but a
   few admin surfaces still needed Fable-style polish and clearer empty/hover states.
2. The biggest visible bug was not in React at all. The Add Father/Mother/Spouse/Son/Daughter form is
   generated by `family-chart` as `.f3-form-cont` / `.f3-form`. The package defaults
   `--background-color` to a dark canvas color, so the add-relative form stayed dark in light mode
   even after the surrounding React UI was cleaned up.

## Fable/public Family reference

Used `https://www.hafizbahtiar.com/family` and the public Family components as the quality reference:

- `FamilyToolbar.tsx`: white card toolbar, slate text, subtle borders, compact controls.
- `PersonDetailPanel.tsx`: clean white/dark panel surfaces, readable hierarchy, restrained cyan
  accents, elegant relationship rows.
- `FamilyListView.tsx` / `PersonSearch.tsx`: selected states, hover states, focus rings, and
  light/dark parity.

The admin builder keeps its editing workflow, but now follows the same design principles: white or
slate-50 light surfaces, slate-900/slate-600 text hierarchy, soft borders, restrained cyan/blue/rose
accents, and paired dark variants.

## Symptoms addressed

- Add-relative form was a dark block in light mode.
- Generated form labels, radio labels, and close/cancel controls were low contrast.
- Generated form inputs used dark backgrounds in light mode.
- Placeholders were too dim.
- Package submit/cancel/delete/remove buttons did not match admin button quality.
- Chart frame had a dark-only border in light mode.
- Relationships section lacked a polished empty state and row affordance.

## Light mode strategy

- Surfaces: `bg-white` and `bg-slate-50`.
- Text: `text-slate-900` for headings and `text-slate-600` / `text-slate-500` for secondary copy.
- Borders: `border-slate-200` / `border-slate-300`.
- Focus: sky/cyan rings for package form controls and existing admin rings for React controls.
- Accents: restrained cyan/blue/rose/amber, matching the public Family pageâ€™s controlled accent use.

## Dark mode preservation

- Dark variants keep deep slate surfaces and readable slate text.
- Existing admin chart canvas remains dark and high contrast.
- Existing admin chart cards and add-relative placeholder cards remain untouched.
- Package form dark mode is now explicit rather than falling through to package defaults.
